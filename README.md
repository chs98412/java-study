# 자바 스터디  
<br><br>
## 대상
 
 
백엔드 직무를 희망하지만 평소 파이썬으로 코테 문제를 풀어서 막상 자바는 어색하거나, 스프링만 끄적끄적해서 객체지향이 뭔지 까먹은 사람  
<br>
 
 
## 목적
 
 
자바를 자바스럽게, 객체지향적으로 코드를 짜는 연습을 한다.  
<br>
 
 
## 필요성
 
 
백엔드 직무는 대부분 스프링 개발자를 뽑기 때문에 (자바공화국) 자바의 이해는 필수적이라 생각됨.

단순 자바 문법이 아닌 “**자바스럽게**” 짜는 법을 공부하려는 이유는 최근 카카오와 같은 대기업에서 2차 코딩테스트로 일반 알고리즘 문제가 아닌 “**API 호출 프로그램**” 프로그램 작성 문제를 내는 경향이 있는데, 코드 작성시 구조를 잘 짜는것이 중요하다는 의견이 많고, 왜 그런 구조를 채택했는지도 추후 면접에서 질문한다고 함. (정보 출처: 구글링 ㅎㅎ,,)
   
<br>
 
## 방식 ( 추후 협의 )

인원: 3~5명

**규칙**

1. 시간 정해놓고 온라인으로 진행 (매일? 주3회?)
2. 벌금 존재 (금액은 협의) (동기부여를 위해)
 
 
**진행 방식**

온라인으로 진행 (디스코드, 구글 미트)

1. 스몰톡으로 시작
2. 각자 공부한 내용 발표 혹은 각자 작성한 코드 발표, 질의
3. 다음 스터디까지 할 내용 결정
- 스터디 전까지 각자 공부한것 혹은 코드 작성한 것 깃에 올리기  
<br>
 
 
## 선택

1. 레퍼런스 도서를 함께 공부한 후, 컨텐츠를 수행 (레퍼런스 도서를 한번씩 정독할 수 있다는 장점, but 노잼)
2. 컨텐츠를 수행하면서 각자 알아서 레퍼런스 도서를 참고 (지루하지도 않고 실전 적용해볼 수 있음, but 레퍼런스 도서를 찬찬히 읽지 않을지도 모름) (후에 필요를 느낄 시 다시 도서들을 정독해도 좋음) (선택 2 선호 ㅎㅎ,,)

레퍼런스 도서 안에 컨텐츠를 수행하는데 필요한 지식들이 있음 (아마도?)  
<br>

> **컨텐츠 진행 방법**: 각자 문제에 해당하는 코드를 짜보고 코드리뷰를 하는 방식으로 진행
**컨텐츠 선정 기준**: 일반 알고리즘처럼 문제해결이 중요한게 아닌 자바스럽게 코드를 짜는것이 목표인 문제들
